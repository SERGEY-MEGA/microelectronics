# ЛР2, вариант 1

## Задача

Считать показания датчика температуры `LM35` с заданной частотой и заданным объемом выборки, сгладить результат методом модифицированного скользящего среднего и запускать сбор по внешнему сигналу.

## Используемые компоненты

- Arduino Uno
- датчик температуры `LM35`
- кнопка или внешний цифровой сигнал для запуска
- макетная плата
- соединительные провода

## Принятые параметры

- частота выборки: `10 Гц`
- объем выборки: `64 отсчета`
- окно модифицированного скользящего среднего: `5`

Если преподаватель дал другие параметры, достаточно изменить константы в файле `lab2_variant1/lab2_variant1.ino`.

## Подключение

### LM35

Если смотреть на плоскую сторону корпуса `LM35`:

- левый вывод -> `+5V`
- средний вывод -> `A0`
- правый вывод -> `GND`

### Сигнал запуска

- один контакт кнопки -> `D2`
- второй контакт кнопки -> `GND`

Используется режим `INPUT_PULLUP`, поэтому внешний резистор для кнопки не нужен.

## Таблица соединений

| Элемент | Контакт элемента | Arduino Uno |
|---|---|---|
| LM35 | VCC | 5V |
| LM35 | VOUT | A0 |
| LM35 | GND | GND |
| Кнопка запуска | 1 | D2 |
| Кнопка запуска | 2 | GND |

## Логика работы

1. После включения контроллер ждет внешний стартовый сигнал.
2. При нажатии кнопки на `D2` начинается серия измерений.
3. Значения температуры сохраняются в массивы:
   - `rawAdcSamples`
   - `rawTemperatureSamplesC`
   - `filteredTemperatureSamplesC`
4. После заполнения выборки массивы остаются в памяти и могут быть просмотрены через отладчик.
5. Повторный внешний сигнал запускает новую серию измерений с перезаписью предыдущих данных.

## Как проверять

1. Соберите схему.
2. Откройте папку `lab2_variant1` в Arduino IDE и загрузите скетч.
3. Откройте монитор порта на `9600 бод`.
4. Подайте стартовый сигнал на `D2`.
5. После завершения выборки посмотрите значения в массивах через отладчик или режим просмотра памяти.
