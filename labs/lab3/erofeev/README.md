# ЛР3, вариант Ерофеева

## Тема

CRC-8 сервер на Arduino Uno с приемом данных по `UART` и выдачей результата по нажатию кнопки.

## Что реализовано

В папке находятся две версии программы:

- `lab3_crc8_reset` - базовая версия, в которой после выдачи CRC по кнопке аккумулятор сбрасывается.
- `lab3_crc8_accumulate` - измененная версия, в которой после выдачи CRC по кнопке аккумулятор не сбрасывается и продолжает накапливать контрольную сумму для последующих сообщений.

## Компоненты

- Arduino Uno
- 1 тактовая кнопка
- соединительные провода
- USB-кабель для связи с ПК

## Подключение

Используется только кнопка и USB-соединение с компьютером.

### Кнопка

- один контакт кнопки -> `D2`
- второй контакт кнопки -> `GND`

В программе включен `INPUT_PULLUP`, поэтому внешний подтягивающий резистор не нужен.

### UART

- передача данных между ПК и Arduino идет через USB-UART преобразователь на плате
- скорость обмена: `9600 бод`

## Таблица соединений

| Элемент | Контакт | Arduino Uno |
|---|---|---|
| Кнопка | вывод 1 | D2 |
| Кнопка | вывод 2 | GND |
| ПК | USB | USB-порт Arduino Uno |

## Быстрая схема

```text
PC <---USB/UART---> Arduino Uno
                     |
                     +-- D2 ---[ button ]--- GND
```

## Важная настройка Serial Monitor

Для корректной проверки нужно выбрать:

- скорость `9600`
- режим отправки: `No line ending` / `Без окончания строки`

Если выбрать `Newline` или `Both NL & CR`, в сообщение будут добавляться дополнительные байты `0x0A` и `0x0D`, из-за чего итоговый CRC изменится.

## Порождающий полином

В обеих версиях по умолчанию используется полином `0x07`, то есть:

`x^8 + x^2 + x + 1`

Если преподаватель выдал другой полином, достаточно изменить константу `CRC_POLYNOMIAL` в нужном `.ino` файле.

## Алгоритм работы

1. Arduino ждет байты из `UART`.
2. Каждый принятый байт сразу включается в вычисление CRC-8.
3. Нажатие кнопки на `D2` выводит текущую контрольную сумму в `Serial Monitor`.
4. В `reset`-версии после этого CRC обнуляется.
5. В `accumulate`-версии после этого CRC не обнуляется и продолжается обработка следующих байтов.

## Проверка задания

### Базовая версия `lab3_crc8_reset`

1. Откройте папку `lab3_crc8_reset` в Arduino IDE.
2. Выберите плату `Arduino Uno`.
3. Загрузите скетч.
4. Откройте `Serial Monitor`, скорость `9600`, режим `No line ending`.
5. Отправьте символ `A`.
6. Нажмите кнопку.
7. Должен появиться результат:

```text
CRC-8 = 0xC0
```

8. Затем отправьте строку `AAA`.
9. Еще раз нажмите кнопку.
10. Должен появиться результат:

```text
CRC-8 = 0x63
```

### Почему `CRC(A)` и `CRC(AAA)` разные

CRC зависит не только от значений байтов, но и от их количества, порядка и промежуточного остатка после каждого шага. Поэтому даже если все символы одинаковые, добавление еще двух байтов `A` меняет весь ход деления по модулю 2 и приводит к другой контрольной сумме.

### Измененная версия `lab3_crc8_accumulate`

1. Откройте папку `lab3_crc8_accumulate` в Arduino IDE.
2. Загрузите скетч.
3. Отправьте `A`.
4. Нажмите кнопку.
5. Получите:

```text
CRC-8 = 0xC0
```

6. После этого отправьте `AAA`.
7. Нажмите кнопку снова.
8. Теперь итог уже считается не для новой строки `AAA`, а для всей накопленной последовательности `AAAA`.
9. Ожидаемый результат:

```text
CRC-8 = 0xEE
```
