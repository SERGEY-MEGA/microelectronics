# Готовые симуляции (Wokwi)

Здесь лежат **готовые пары файлов** для симулятора [Wokwi](https://wokwi.com) в браузере (Mac, Windows, Linux). Отдельный Proteus и Wine **не нужны**: схема и код лежат в репозитории как текст.

В каждой папке:

- `diagram.json` — схема (Arduino Uno, провода, кнопки, светодиоды и т.д.)
- `sketch.ino` — тот же скетч, что и в `labs/...` (копия для удобной загрузки в Wokwi)

## Папки

| Папка | Содержание |
|-------|------------|
| `lab1_variant1` | ЛР1 вар.1: 6 бит на Port B, кнопки D2/D3 |
| `lab1_variant2` | ЛР1 вар.2: бегущий огонь, кнопки D2/D3 |
| `lab2_variant1` | ЛР2 вар.1: «LM35» на A0, старт по D2 (см. примечание ниже) |
| `lab2_variant2` | ЛР2 вар.2: потенциометр A0, кнопка D2, Serial |
| `lab3_crc8_reset` | ЛР3: CRC-8, сброс после отчёта |
| `lab3_crc8_accumulate` | ЛР3: CRC-8 без сброса |
| `lab4_variant2` | ЛР4 вар.2: фоторезистор, 8 LED D6–D13 |

## Как открыть в браузере (показать с экрана)

1. Зайдите на [wokwi.com](https://wokwi.com) и войдите в аккаунт (бесплатная регистрация удобна, чтобы сохранять проекты).
2. Нажмите **New Project** и выберите **Arduino Uno** (или откройте любой пустой проект с платой Arduino Uno).
3. Откройте вкладку **diagram.json** (или редактор схемы → режим кода/JSON, в зависимости от интерфейса: иногда это меню **Diagram → Code**).
4. **Полностью замените** содержимое на текст из файла `diagram.json` нужной папки (скопируйте из репозитория или с диска).
5. Откройте вкладку с кодом (например `sketch.ino` / **Code**) и **вставьте** содержимое локального `sketch.ino` из той же папки.
6. Нажмите **Start Simulator** (зелёная кнопка). Кнопки на схеме можно нажимать мышью; у части кнопок заданы горячие клавиши на схеме.

Подробнее о формате схемы: [diagram.json в документации Wokwi](https://docs.wokwi.com/diagram-format).

## ЛР3 и Serial Monitor

В коде для UART указано **9600 бод** и режим ввода **без перевода строки** (`No line ending`), как в скетче. В `diagram.json` для ЛР3 включено `"newline": "none"`, чтобы при отправке символов не добавлялся лишний `\n`.

## ЛР2 вариант 1 и «LM35»

В Wokwi на схеме стоит **потенциометр** на `A0`: он задаёт напряжение, как при работе с реальным датчиком. Для отчёта и демонстрации алгоритма этого достаточно; физический датчик LM35 на макетке по-прежнему подключается по `labs/lab2/variant1/README.md`.

## VS Code

Если установлено расширение **Wokwi for VS Code**, можно открыть **папку** с `diagram.json` и `sketch.ino`, собрать прошивку под Arduino Uno и запустить симулятор из редактора. Потребуется настроенная среда сборки (например `arduino-cli`). Для большинства задач достаточно браузерной версии.

## Обновление файлов после правок в `labs/`

Из корня репозитория:

```bash
python3 scripts/build_wokwi_simulations.py
```

Скрипт перезапишет `diagram.json` и скопирует актуальные `sketch.ino` из `labs/`.

## Почему не Proteus `.pdsprj`

Готовые файлы проекта Proteus — проприетарный формат, их нельзя надёжно сгенерировать из репозитория без установленного Proteus. Набор **JSON + INO** для Wokwi воспроизводим, хранится в Git и открывается на Mac без эмулятора Windows.
