# Краткая выжимка по заданию

## ЛР1. Ввод-вывод дискретных сигналов

### Вариант 1

Реализовать через `Port B` вывод 6-битного числа в двоичном коде на светодиодной линейке. Инкрементирование и декрементирование выполнять по нажатию тактовых кнопок.

### Вариант 2

Реализовать бегущий огонь на светодиодной линейке из 6 светодиодов. Увеличение и уменьшение скорости выполнять по нажатию тактовых кнопок.

## ЛР2. Регистрация показаний аналоговых датчиков

### Вариант 1

Снимать показания аналогового датчика температуры `LM35` с заданной частотой и заданным объемом выборки. Выполнить сглаживание методом модифицированного скользящего среднего. Запуск выборки выполнять по внешнему сигналу. Выборку выгружать из памяти микроконтроллера через отладчик.

### Вариант 2

Разработать систему дискретного сбора и статистической обработки данных для потенциометра. Считывание выполнять по нажатию кнопки с программным антидребезгом. На скользящем интервале из десяти последних измерений, организованном кольцевым буфером, вычислять среднее арифметическое и среднеквадратичное отклонение. Выгрузку данных выполнять в монитор порта по запросу пользователя.

## ЛР4. Система управления с обратной связью

### Вариант 2

Индикатор уровня освещенности: количество светящихся светодиодов на панели из 8 светодиодов должно зависеть от освещенности фоторезистора `LDR GL5516`. Аналоговый сигнал считывается с `A0`, светодиоды подключаются к выводам `D6-D13` через токоограничивающие резисторы.

## ЛР3. CRC-8 сервер по UART

### Вариант Ерофеева

Реализовать сервер на базе Arduino Uno, принимающий сообщения от ПК через интерфейс `UART` со скоростью `9600 бод` и вычисляющий для них контрольную сумму `CRC-8` по заданному порождающему полиному. Выдачу рассчитанной контрольной суммы выполнять по нажатию кнопки. Кнопка подключается с использованием внутреннего подтягивающего резистора `INPUT_PULLUP`.

В отчете требуется:

- пояснить, что такое порождающий полином;
- описать алгоритм вычисления CRC;
- показать, зачем используется CRC;
- проверить передачу одного символа `A`;
- проверить передачу строки `AAA`;
- объяснить, почему при изменении длины сообщения меняется CRC;
- модифицировать программу так, чтобы после нажатия кнопки CRC не обнулялся, а продолжал накапливаться.

## Принятые допущения

- Для ЛР2, вариант 1 выбраны параметры по умолчанию: `10 Гц` и `64 отсчета`.
- Для схем с кнопками используется `INPUT_PULLUP`, поэтому кнопка замыкается на `GND`.
- Для всех светодиодов предполагается использование резисторов `220 Ом`.
- Для ЛР4 принят типовой делитель: `+5V -> LDR -> A0 -> 10 кОм -> GND`.
- Для ЛР3 принят стандартный полином CRC-8 `0x07`, то есть `x^8 + x^2 + x + 1`.
